## 概要

β フェーズの検証目標（KPI）を計測できるよう、主要ユーザー行動に対する Analytics イベントを定義・実装する。

## 計測対象 KPI（β）

| 指標 | 目標値 |
|------|--------|
| 初回完了率 | 70% 以上 |
| D1 継続率 | 40% 以上 |
| D7 継続率 | 20% 以上 |
| 7日ストリーク到達率 | 15% 以上 |
| ストリーク途切れ後復帰率 | 30% 以上 |
| クラッシュフリー率 | 99.5% 以上 |

## イベント一覧（案）

| イベント名 | 発火タイミング | パラメータ例 |
|------------|----------------|--------------|
| `onboarding_start` | オンボーディング開始 | — |
| `onboarding_complete` | 初回プランク完了 | `plank_type_id` |
| `plank_complete` | プランク完了 | `plank_type_id`, `seconds`, `earned_exp`, `streak_days` |
| `streak_broken` | ストリーク途切れ検知 | `previous_streak`, `longest_streak` |
| `streak_milestone` | マイルストーン到達 | `milestone_days` |
| `level_up` | レベルアップ | `new_level`, `abs_stage` |
| `degeneration` | 腹筋退化 | `new_abs_stage` |
| `app_open` | アプリ起動 | `days_since_last_open` |

## スコープ

### 含む

- 上記イベントの実装
- イベント名・パラメータのドキュメント（`tickets/beta/` または `01-design/` に追記）
- Firebase コンソールでのファネル・オーディエンス設定（手順書）

### 含まない

- 外部 BI ツール連携
- 個人を特定できるパラメータの送信

## 受け入れ条件

- [ ] β KPI に必要なイベントがすべて実装されている
- [ ] D1 / D7 継続率を Firebase で算出できる（またはエクスポート手順がある）
- [ ] 7日ストリーク到達率を計測できる
- [ ] ストリーク途切れ後3日以内の復帰率を計測できる
- [ ] イベント定義書が作成されている
