## 概要

10〜30名のクローズド β テスターに TestFlight（または同等）で配信し、7日以上の実地利用とフィードバックを収集する。

## スコープ

### 含む

- テスター募集・オンボーディング手順の整備
- TestFlight / 内部テストトラックへのビルド配信
- フィードバック収集手段（Google Form、アンケート、Discord 等）
- 収集項目の定義
  - ホーム統合 UI（キャラビュー⇔種目ビュー、スタートまでの導線）の分かりやすさ
  - ストリーク +1 演出の気持ちよさ
  - 退化の厳しさ
  - Pro ロック UI の印象（種目カルーセル）
  - バグ・クラッシュ報告
- クラッシュレポートの監視（Crashlytics 任意導入）

### 含まない

- ストア公開（BETA-018）
- 有償マーケティング

## 受け入れ条件

- [ ] 10名以上がアプリをインストールし利用開始した
- [ ] 7日以上利用したテスターが複数名いる
- [ ] 構造化されたフィードバック（アンケート等）を収集した
- [ ] 重大バグ・クラッシュが記録・トリアージされている
- [ ] β DoD「10名以上が7日以上利用し、フィードバックを収集した」を満たす

## 備考

MVP → β の Go 条件：オンボーディング〜1日ループが動作、社内7日連続利用テスト可能であること。
