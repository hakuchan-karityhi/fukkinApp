# [BETA-016] 数値バランス調整（プレイテスト反映）

## メタ情報

| 項目 | 値 |
|------|-----|
| フェーズ | β |
| 要件ID | PL-15（関連） |
| 優先度 | Must |
| 依存 | BETA-001, BETA-011 |
| 参照 | `03-phase.md` §4.3 / `02-youken.md` §6 |

## 概要

クローズド β のプレイテスト結果と KPI に基づき、ゲーム数値を **1回以上** 調整する。退化の厳しさ・成長速度・ストリーク倍率などが主な調整対象。

## 調整対象

| 項目 | 現行たたき台 | 調整の観点 |
|------|--------------|------------|
| EXP 減少量 | 2日目: max(10, floor(基本×0.5))、3日目以降: 基本EXP/日 | サボりすぎない・戻しやすい |
| レベル閾値 | 150 / 400 / 800 / 1,400 | 1〜2週でステージ1到達感 |
| ストリーク倍率 | 1.0〜1.5 | 続けるモチベーション |
| 退化タイミング | 前ステージ閾値を下回ったら1段階 | 離脱率とのバランス |

## スコープ

### 含む

- プレイテストフィードバックの整理（退化が厳しすぎ/緩すぎ等）
- Remote Config 経由での数値変更とリリース
- 変更前後の KPI 比較（BETA-011）
- 調整内容の記録（変更理由・日付・値）

### 含まない

- ゲームルール自体の変更（例: ストリーク締切のカスタム化）

## 受け入れ条件

- [ ] プレイテストから数値に関するフィードバックを収集した
- [ ] Remote Config で数値を1回以上変更し、ユーザーに反映した
- [ ] 退化後復帰率・D7 継続率の変化を確認した
- [ ] 調整ログがドキュメント化されている
- [ ] β DoD「数値調整を1回以上反映した」を満たす
