# [BETA-008] 同日複数回プランク（2回目 50% EXP）

| 項目 | 値 |
|------|-----|
| フェーズ | β |
| 要件ID | PL-19 |
| 優先度 | Must |
| 依存 | MVP 完了 |
| 参照 | `02-youken.md` §3.1.2, §3.3.1 / `04-basic-design.md` §2.1 |

## 概要

同一カレンダー日に2回目以降のプランクを実施した場合、基本 EXP を 50%（切り捨て）に減算する。1日の獲得 EXP 上限は 150。MVP では任意（△）だったが、β では Must として実装する。

## スコープ

### 含む

- `ExpCalculator` の同日2回目減算ロジック
- `WorkoutRecord.sessionIndexOfDay` の正しいインクリメント
- 結果画面での内訳表示（2回目であることの明示）
- ホーム種目ビューの「完了時 EXP（見込み）」が同日2回目・日次上限を反映してリアルタイム更新される
- 日次 EXP 上限（150）の適用
- ストリークは同日複数回でも +1 のみ（既存ルール維持）

### 含まない

- 3回目以降の追加ペナルティ（2回目以降は同率 50% で統一）

## 受け入れ条件

- [ ] 同日1回目はフル EXP、2回目以降は基本 EXP × 0.5（切り捨て）が適用される
- [ ] 1日合計 150 EXP を超えない
- [ ] ストリークは1日1回のみ +1 される
- [ ] 日付跨ぎ後は sessionIndexOfDay がリセットされる
- [ ] 1回目完了後、ホームに戻ると EXP 見込みが2回目用（50%）に更新される

## 実装メモ

- `CompletePlankUseCase` が当日の実施回数を取得して `isSecondSessionToday` を渡す（`04-basic-design.md` §6.2）

## 備考

基本設計では MVP 向けに記載があるが、フェーズ計画では β で Must。
