# [BETA-008] 同日複数回プランク（2回目 再実施ボーナス）

## メタ情報

| 項目 | 値 |
|------|-----|
| フェーズ | β |
| 要件ID | PL-19 |
| 優先度 | Must |
| 依存 | MVP 完了 |
| 参照 | `02-youken.md` §3.1.2, §3.3.1 / `04-basic-design.md` §2.1 |

## 概要

同一カレンダー日の **2回目** のプランク完了時のみ、獲得 EXP に **+15%（四捨五入）** の再実施ボーナスを付与する。1回目・3回目以降は通常 EXP。1日の獲得 EXP 上限は 150 を維持する。MVP では任意（△）だったが、β では Must として実装する。

## スコープ

### 含む

- `ExpCalculator` の同日2回目ボーナスロジック（`sessionIndexOfDay == 2` のときのみ適用）
- `WorkoutRecord.sessionIndexOfDay` の正しいインクリメント
- 結果画面での内訳表示（2回目ボーナスであることの明示）
- ホーム種目ビューの「完了時 EXP（見込み）」が同日2回目・日次上限を反映してリアルタイム更新される
- 日次 EXP 上限（150）の適用
- ストリークは同日複数回でも +1 のみ（既存ルール維持）
- Remote Config キー `second_session_exp_bonus_rate`（デフォルト 1.15、チューニング想定 1.10〜1.20）

### 含まない

- 3回目以降の追加ボーナス・減算（3回目以降は通常 EXP のまま）
- 日次 EXP 上限の引き上げ

## 受け入れ条件

- [ ] 同日1回目はフル EXP（ボーナスなし）
- [ ] 同日2回目のみ `round(基本 EXP × 難易度倍率 × ストリーク倍率 × 1.15)` が適用される
- [ ] 同日3回目以降は1回目と同様にボーナスなしの通常 EXP
- [ ] 1日合計 150 EXP を超えない（上限到達分は切り捨て）
- [ ] ストリークは1日1回のみ +1 される
- [ ] 日付跨ぎ後は sessionIndexOfDay がリセットされる
- [ ] 1回目完了後、ホームに戻ると EXP 見込みが2回目用（+15% ボーナス込み）に更新される
- [ ] 3回目完了後、ホームの EXP 見込みは通常（ボーナスなし）に戻る

## 実装メモ

- `CompletePlankUseCase` が当日の実施回数を取得し `sessionIndexOfDay` を渡す（`04-basic-design.md` §6.2）
- 旧仕様の `second_session_exp_rate`（0.5 減算）は `second_session_exp_bonus_rate`（1.15 倍）に置き換える
- 日次上限適用は各セッション付与前に「本日獲得済み EXP」を参照し、超過分を 0 にクランプ

## 備考

- 再実施ボーナスは「頑張ったね」感を出しつつ、1回目を主役に保つため **2回目限定**
- 基本設計では MVP 向けに記載があるが、フェーズ計画では β で Must
