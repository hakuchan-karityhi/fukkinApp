## 概要

β フェーズで数値バランスをアプリ再リリースなしで調整できるよう、ゲーム定数を Firebase Remote Config（または同等）へ外部化する。MVP ではコード内定数だった倍率テーブル・レベル閾値・ペナルティ係数を対象とする。

## スコープ

### 含む

- ストリーク倍率テーブル
- レベル・腹筋ステージ閾値（150 / 400 / 800 / 1,400）
- EXP 減少・ペナルティ係数
- 再実施ボーナス加算率（step 0.05）・上限倍率（cap 1.15）、日次 EXP 上限（150）
- フォールバック：Remote Config 未取得時はアプリ内蔵のデフォルト値を使用

### 含まない

- プランク種目マスタ（`plank_types.json` はアプリ内蔵のまま）
- マイルストーン日数の動的変更（任意。余力があれば対応）

## 受け入れ条件

- [ ] Remote Config からゲーム定数を読み込み、Domain サービスが参照できる
- [ ] オフライン時・取得失敗時にデフォルト値で正常動作する
- [ ] 定数変更後に EXP 計算・退化・ストリーク倍率が期待どおり変わることを確認した
- [ ] 変更履歴または設定キー一覧をドキュメント化した

## 実装メモ

- `ExpCalculator`, `PenaltyService`, `LevelService`, `StreakService` が参照する定数を一箇所に集約
- β では Remote Config の更新を手動で行い、週次チューニングに使う
