## 概要

記録タブを **カレンダー ＋ ストリーク ＋ 日別実施履歴** の構成に整理する。カレンダーで日付をタップすると、その日に実施したプランクの記録をカレンダー下に一覧表示する。

## スコープ

### 含む

- 月間カレンダー（実施日マーク）に **日付タップ** で選択状態
- **現在のストリーク**・**最長ストリーク**の常時表示
- 選択日の `WorkoutRecord` 一覧（`WorkoutRepository.getByDate`）
  - 種目名・目標秒数・獲得 EXP・完了時刻
  - 同日複数回の場合は `sessionIndexOfDay` を表示（任意）
- 未選択時は **今日** を初期選択
- 実施なし日は空状態メッセージ（例：「この日は記録がありません」）
- 月切り替え時も選択日の履歴を更新

### 含まない

- 週次/月次レポート（Pro / PL-26）
- 種目別合計秒数の集計グラフ
- カレンダー以外の称号・マイルストーン編集 UI の大幅変更（既存があれば維持可）

## 受け入れ条件

- [ ] 記録タブにカレンダーとストリーク（現在・最長）が表示される
- [ ] カレンダーの日付をタップすると選択状態になる
- [ ] 選択日のプランク実施履歴がカレンダー下に一覧表示される
- [ ] 同日に複数回実施した場合、すべての記録が表示される
- [ ] 未実施日は空状態が表示される
- [ ] アプリ再起動後も履歴が正しく表示される

## 実装メモ

- `MonthCalendar` に `selectedDate` / `onDateSelected` を追加
- 記録一覧 Widget を `records_screen.dart` または `widgets/day_workout_list.dart` に分離
- Provider: `getByDate(selectedDate)` を `FutureProvider.family` で取得

## 備考

既存の記録画面にマイルストーン目標編集等がある場合、本チケットは **カレンダー＋ストリーク＋日別履歴** を主 UI とし、副次機能の配置は実装時に整理してよい。
