# [BETA-008] 同日複数回プランク（再実施ボーナス段階加算）

## メタ情報

| 項目 | 値 |
|------|-----|
| フェーズ | MVP |
| 要件ID | PL-19 |
| 優先度 | Must |
| 依存 | MVP 完了 |
| 参照 | `02-youken.md` §3.1.2, §3.3.1 / `04-basic-design.md` §2.1 |

## 概要

同一カレンダー日の **2回目以降** のプランク完了時、回数に応じて再実施ボーナスを **+5% ずつ** 付与する（2回目 +5%、3回目 +10%、4回目以降 +15% でキャップ）。1回目は通常 EXP。1日の獲得 EXP 上限は 150 を維持する。MVP では任意（△）だったが、β では Must として実装する。

## スコープ

### 含む

- `ExpCalculator` の再実施ボーナス段階加算ロジック（`sessionIndexOfDay` に応じた倍率）
- `WorkoutRecord.sessionIndexOfDay` の正しいインクリメント
- 結果画面での内訳表示（回数・ボーナス率の明示。例：「再実施（3回目 +10%）」）
- ホーム種目ビューの「完了時 EXP（見込み）」が同日回数・日次上限を反映してリアルタイム更新される
- 日次 EXP 上限（150）の適用
- ストリークは同日複数回でも +1 のみ（既存ルール維持）
- Remote Config キー `repeat_session_bonus_step`（デフォルト 0.05）、`repeat_session_bonus_cap`（デフォルト 1.15）

### 含まない

- 日次 EXP 上限の引き上げ
- 4回目以降の追加キャップ引き上げ（+15% 固定）

## 再実施ボーナス倍率

```
再実施ボーナス倍率 =
  sessionIndexOfDay == 1  → 1.0
  sessionIndexOfDay >= 2  → min(1.0 + step × (sessionIndexOfDay - 1), cap)

デフォルト: step = 0.05, cap = 1.15
```

| 回 | 倍率 | ボーナス |
|----|------|----------|
| 1回目 | 1.00 | なし |
| 2回目 | 1.05 | +5% |
| 3回目 | 1.10 | +10% |
| 4回目以降 | 1.15 | +15%（キャップ） |

## 受け入れ条件

- [ ] 同日1回目はフル EXP（ボーナスなし）
- [ ] 同日2回目は +5%、3回目は +10%、4回目以降は +15%（四捨五入）が適用される
- [ ] 1日合計 150 EXP を超えない（上限到達分は切り捨て）
- [ ] ストリークは1日1回のみ +1 される
- [ ] 日付跨ぎ後は sessionIndexOfDay がリセットされる
- [ ] 1回目完了後、ホームに戻ると EXP 見込みが2回目用（+5%）に更新される
- [ ] 3回目完了後、ホームの EXP 見込みが +10% ボーナス込みに更新される

## 実装メモ

- `CompletePlankUseCase` が当日の実施回数を取得し `sessionIndexOfDay` を渡す（`04-basic-design.md` §6.2）
- 旧仕様の `second_session_exp_rate`（0.5 減算）・`second_session_exp_bonus_rate`（1.15 固定）は `repeat_session_bonus_step` / `repeat_session_bonus_cap` に置き換える
- 日次上限適用は各セッション付与前に「本日獲得済み EXP」を参照し、超過分を 0 にクランプ
- `ExpCalculator.getRepeatSessionMultiplier(sessionIndexOfDay)` のような純関数で倍率を算出

## 備考

- 追加トレーニングへのモチベーションを高めつつ、+15% キャップと日次 150 上限で刷り込みを抑制
- 基本設計では MVP 向けに記載があるが、フェーズ計画では β で Must
