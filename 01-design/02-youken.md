## 1. 目的

ストリークを中心に据えたプランク習慣化アプリを開発し、ユーザーが毎日短時間のプランクを継続できるようにする。キャラクターとの同期プランク体験、EXP・腹筋進化、ストリーク倍率により「続けるほど育つ」好循環を実現する。

### 1.1 ビジネス目的

| # | 目的 |
|---|------|
| 1 | ストリーク習慣の定着（連続日数の可視化と維持動機） |
| 2 | キャラ育成による継続動機づけ（腹筋進化・退化） |
| 3 | ストリークと育成の相乗効果（倍率加速） |
| 4 | サボりへの適度なペナルティと前向きな復帰体験 |
| 5 | MVP でコア体験を検証し、正式版でフリーミアム＋Pro を展開 |

### 1.2 対象ユーザー

- **メイン:** 20〜40代、短時間で運動習慣をつけたいデスクワーカー・在宅ワーカー
- **サブ:** プランク初心者、ストリーク型アプリ経験者
- **初期対象外:** 本格筋トレ管理・食事管理を主目的とするユーザー

#### 3.1.2 ゲーム数値仕様

**EXP 計算**

```
獲得 EXP = round(基本 EXP × 難易度倍率 × ストリーク倍率)
基本 EXP = ユーザー設定の目標秒数（1秒 = 1 EXP）
```

**ストリーク倍率**

| 連続日数 | 倍率 |
|----------|------|
| 1〜2日 | 1.0 |
| 3〜6日 | 1.1 |
| 7〜13日 | 1.2 |
| 14〜29日 | 1.3 |
| 30日以上 | 1.5 |

**マイルストーン**

| 日数 | 称号 |
|------|------|
| 3 | 習慣の芽 |
| 7 | 1週間の相棒 |
| 14 | 2週間マスター |
| 30 | 継続の証 |
| 60 | 習慣の達人 |
| 100 | レジェンド |

**レベル・腹筋ステージ**

| レベル帯 | 累計 EXP | 腹筋ステージ |
|----------|----------|--------------|
| Lv.1〜3 | 0〜150 | 0 |
| Lv.4〜7 | 150〜400 | 1 |
| Lv.8〜12 | 400〜800 | 2 |
| Lv.13〜18 | 800〜1,400 | 3 |
| Lv.19+ | 1,400+ | 4 |

**EXP 減少・退化**

- 最終実施日から2日目終了時：`-max(10, floor(前回基本 EXP × 0.5))`
- 3日目以降、毎日：`-前回基本 EXP`（下限は現在ステージの最低 EXP）
- 前ステージ閾値を下回ったら腹筋ステージ1段階退化

**ストリークルール**

- 1日の定義：カレンダー日（0:00〜23:59）に1回以上プランク完了
- 同日のストリーク倍率は当日開始時点で固定
- 途切れ時：現在ストリーク 0、倍率 1.0。最長ストリークは保持

## 4. 非機能要件

| カテゴリ | 要件内容 |
|----------|----------|
| パフォーマンス | ホーム表示 2秒以内。タイマー誤差 ±1秒以内（バックグラウンド復帰時も補正） |
| 可用性 | オフラインでプランク実施・EXP 付与・ストリーク更新が可能（MVP） |
| 信頼性 | クラッシュ・タイマー不具合は最優先修正。進捗データの消失を防ぐ |
| セキュリティ | 個人情報最小化。MVP は端末ローカルのみ。クラウド同期時は通信暗号化 |
| プライバシー | カメラ・位置情報は MVP では不使用 |
| 互換性 | iOS / Android。MVP はどちらか一方から開始（要決定） |
| ユーザビリティ | ストリーク数字が一目でわかる。通知文言は責めず「守ろう」トーン |
| 保守性 | プランク種目・倍率・マイルストーンはマスタデータ化し、コード変更なしで調整可能に |
| アクセシビリティ | タイマー中の視認性（コントラスト・文字サイズ）。将来 VoiceOver 対応検討 |
| 課金公平性 | Pay to Win 禁止。腹筋ステージ・EXP 倍率の直接購入不可。Pro のセット完了ボーナスなし |

## 6. 運用・更新方針

| 項目 | 方針 |
|------|------|
| 数値チューニング | 週次で継続率・ストリーク到達率を確認し、EXP 減少・倍率を小さく調整 |
| 通知文案 | ストリーク日数を含める。「責める」より「待ってる・守ろう」トーンで A/B テスト |
| 季節イベント | 新年・夏前等で限定セリフ・キャンペーン（EXP 倍率 UP は慎重に） |
| コンテンツ追加 | 新プランク種目は Pro 側に追加。無料2種は固定 |
| 障害対応 | タイマー・データ消失系は最優先。習慣アプリの信頼が土台 |
| レビュー対応 | 「課金で強くなる」印象のフィードバックは Pro 設計を見直し |

## 9. 成功指標（KPI）

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| D1 継続率 | 40% 以上 | 初回完了ユーザーの翌日再訪 |
| D7 継続率 | 20% 以上 | 7日後も1回以上実施 |
| 7日ストリーク到達率 | 15% 以上 | 初回完了から7日以内にストリーク7 |
| 30日ストリーク到達率 | 5% 以上 | 初回完了から30日以内にストリーク30 |
| 平均ストリーク長 | 5日 以上 | アクティブユーザーの現在ストリーク平均 |
| 最長ストリーク更新率 | 10% 以上 | 月1回以上、過去最高を更新したユーザー割合 |
| ストリーク途切れ後復帰率 | 40% 以上 | 途切れ後3日以内に再実施 |
| 初回完了率 | 70% 以上 | オンボーディング開始 → 初回プランク完了 |
| 退化後復帰率 | 30% 以上 | EXP 減少後7日以内に再実施 |
| Pro 転換率（v1.0） | 要設定 | 無料ユーザーからのサブスク開始率 |

## 10. リスク・懸念事項

| リスク内容 | 影響 | 対策 |
|------------|------|------|
| サボりペナルティで離脱 | 継続率低下 | 退化は段階的、永久喪失なし、復帰演出は前向きに |
| ストリークプレッシャーで離脱 | D7 低下 | 途切れ演出は責めない、最長記録は残す、復帰を簡単に |
| Pro 種目ロックが Pay to Win 印象 | レビュー悪化 | 無料2種で毎日育成完結を明示。高倍率＝Pro はコンテンツ課金と説明 |
| キャラアニメ工数 | 開発遅延 | MVP は静止画＋簡易アニメ。種目は段階追加 |
| タイマー不具合 | 信頼喪失 | 最優先修正。バックグラウンド・割り込み時のテスト強化 |
| 数値バランス | 成長が早すぎ/遅すぎ | β でプレイテスト。Remote Config で調整可能に |
| 途中終了の扱い未決 | 実装ブレ | 要件確定前に「設定秒数未達=0 EXP」or「実測秒数」を決定 |

## 11. 制約条件

| 項目 | 内容 |
|------|------|
| スコープ | MVP では課金・広告・クラウド同期を含めない |
| プラットフォーム | MVP は iOS or Android の一方から（要決定） |
| Pay to Win | 腹筋ステージ・EXP 倍率の直接購入、セット完了ボーナスは不可 |
| 広告 | プランク実行中・結果演出中は広告禁止。動画広告は **演出完了後のみ** 表示 |
| データ | MVP は個人情報を収集しない（分析イベントのみ） |
| 種目 | プランク以外の運動は v1.0 スコープ外（将来拡張） |
| 無料種目 | ベーシック・ニープランクの2種は固定（変更時は企画書と同期） |
