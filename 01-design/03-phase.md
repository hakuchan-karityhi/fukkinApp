## 1. 目的

本書はふっきんの開発を段階的に分割し、各フェーズでの **目的・実装範囲・検証目標・リリース条件** を明確化することを目的とする。  
企画書・要件定義書で定義した「ストリーク中心の習慣化体験」を、**MVP で検証 → β で数値調整 → 正式版で収益化** する流れで進める。

### フェーズ間の判断基準

| 遷移 | Go の条件 |
|------|-----------|
| 準備 → MVP 開発 | 未決事項のうち実装ブロッカー（途中終了ルール、プラットフォーム先行）が確定している |
| MVP → β | オンボーディング〜1日ループが動作。クラッシュなし。社内で7日連続利用テスト可能 |
| β → 正式版 | D7 継続率・7日ストリーク到達率が目標の80%以上。数値バランスが許容範囲 |
| 正式版リリース | 課金・広告のストア審査通過。Pro 機能の E2E テスト完了。**EU 向け GDPR 対応完了** |
| β → ソフトローンチ（EU 含む） | プライバシーポリシー草案、分析の同意フロー（オプトイン）が実装済み |

## 4. フェーズ別詳細

### 4.1 準備フェーズ（第1〜3週）

| 項目 | 内容 |
|------|------|
| 目的 | 実装前にデザイン・未決事項を固め、MVP 開発の手戻りを減らす |
| 成果物 | キャララフ、腹筋5段階、ストリーク中心ホームのワイヤー、主要画面モック |
| 確定すること | iOS/Android 先行、途中終了の EXP ルール、キャラビジュアル方向 |
| スコープ外 | 実装・ストア申請 |
| **GDPR** | プライバシー方針のたたき台作成（データ一覧の洗い出し）。個人を特定しない設計を確認 |

### 4.2 MVP版（第4〜11週・6〜8週開発）

| 項目 | 内容 |
|------|------|
| 目的 | **「ストリーク × プランク × キャラ育成」** のコアループが成立するか検証する |
| リリース形態 | TestFlight / 内部テスト配信。課金・広告なし |
| 実装する画面 | スプラッシュ、オンボーディング、ホーム、種目選択、秒数設定、プランク実行、結果、設定 |
| プランク種目 | **ベーシック・ニープランクの2種のみ** |
| キャラ表現 | 静止画＋簡易アニメ（Lottie 等）。腹筋5段階の差分 |
| ゲーム数値 | 企画書のたたき台をそのまま実装。Remote Config は任意 |
| 検証すること | 初回完了率、D1 継続、ストリーク +1 演出の気持ちよさ、退化の厳しさ |
| スコープ外 | Pro、広告、クラウド同期、9種目、マイルーティン |
| **GDPR** | ローカルのみ・個人情報非収集を原則。設定に「全データ削除」。Analytics 導入時は同意なしで送信しない |

**MVP の完了定義（DoD）**

- [ ] 初回オンボーディングから初回プランク完了まで一連で動作する
- [ ] ストリークがホーム最上位に表示され、完了で +1 される
- [ ] EXP が秒数 × 倍率で計算され、内訳が結果画面に表示される
- [ ] 未実施日でストリーク途切れ・EXP 減少・退化が動作する
- [ ] データが端末再起動後も保持される
- [ ] プランク中にクラッシュ・タイマー停止がない
- [ ] 設定からローカルデータを完全削除できる

### 4.3 β版（第12〜16週・2〜4週）

| 項目 | 内容 |
|------|------|
| 目的 | 数値バランス調整、継続率の実地検証、種目・演出の拡充 |
| リリース形態 | クローズド β（10〜30名）→ ソフトローンチ（限定地域・限定マーケ） |
| プランク種目 | **5〜9種** へ段階拡張。キャラポーズ追加 |
| 追加機能 | ストリーク通知、分析イベント、マイルストーン 60/100日、キャラセリフ拡充 |
| 調整対象 | EXP 減少量、レベル閾値、ストリーク倍率、退化タイミング |
| Pro 関連 | 9種一覧＋ロック UI のモック（課金なし）。Pro 誘導画面のデザイン確認 |
| スコープ外 | 本番課金、動画広告、クラウド同期 |
| **GDPR** | プライバシーポリシー公開（Web + アプリ内リンク）。分析 ON 時は **明示的オプトイン** 同意画面 |

**β の完了定義（DoD）**

- [ ] 10名以上が7日以上利用し、フィードバックを収集した
- [ ] D7 継続率・7日ストリーク到達率を計測した
- [ ] 数値調整を1回以上反映した
- [ ] ストリーク途切れ後の復帰率を確認した
- [ ] ソフトローンチでストア公開（限定）し、クラッシュ率を確認した
- [ ] EU ユーザーを含む場合、分析の同意フローが動作する（未同意時は計測しない）

### 4.4 正式版 v1.0（第17週〜）

| 項目 | 内容 |
|------|------|
| 目的 | フリーミアムモデルでの本番運用開始。収益化と Pro 価値の検証 |
| リリース形態 | App Store / Google Play 本番公開 |
| 無料プラン | 2種目、完了後動画広告、育成・ストリークすべて |
| Pro プラン | 全9種、広告非表示、マイルーティン、マイ種目、クラウド同期、詳細レポート、ストリークフリーズ |
| 新規画面 | Pro 誘導、マイルーティン一覧/編集、詳細レポート、動画広告 |
| 運用開始 | 週次 KPI 確認、通知文案 A/B、クラッシュ監視 |
| **GDPR** | 本番用プライバシーポリシー・利用規約、広告 CMP（UMP）、クラウドデータ削除・エクスポート、DPA 確認 |

**v1.0 の完了定義（DoD）**

- [ ] 無料・Pro の機能切り分けが要件どおり動作する
- [ ] 完了後動画広告が演出の後のみ表示される（Pro はスキップ）
- [ ] サブスク購入・復元・解約後の権限変化が正しい
- [ ] ストア審査・プライバシーポリシー・利用規約が整備されている
- [ ] Pro 転換率の計測ができる
- [ ] EU 配信地域で GDPR チェックリスト（§10）をすべて満たしている
- [ ] 広告表示前に EU ユーザーへ同意取得（UMP / CMP）が完了している
- [ ] Pro ユーザーのアカウント削除・データ消去リクエストに対応できる

## 8. リスクとフェーズ別対策

| リスク | 影響フェーズ | 対策 |
|--------|--------------|------|
| 未決事項が残ったまま MVP 着手 | 準備 | 途中終了ルール・プラットフォームを準備フェーズで必ず確定 |
| キャラアニメ工数超過 | MVP | 2種・静止画中心。β で段階追加 |
| 退化が厳しすぎて β で離脱 | β | プレイテストで EXP 減少量を調整。Remote Config 化 |
| ストリーク演出が弱い | MVP | MVP 内で +1 演出を最優先品質に。β 前にユーザーテスト |
| 課金・広告の審査リジェクト | 正式版 | 早めにストアガイドライン確認。テスト環境で課金検証 |
| 完了後広告で離脱増 | 正式版 | 広告は演出後のみ。Pro 転換導線を結果画面直後に配置 |
| GDPR 違反・ストア削除 | β / 正式版 | 同意なし計測・広告を実装しない。法務レビュー or テンプレートポリシー利用 |
| クラウド同期のデータ漏洩 | 正式版 | 通信暗号化、最小データ保存、削除 API、DPA 締結 |
| 子どもデータの誤収集 | 全フェーズ | 年齢ゲートは設けない方針ならストアで 13+ / 16+ を明示 |

## 9. GDPR 対応計画（EU 配信）

### 9.1 ふっきんで扱うデータの分類

| データ | 例 | 個人データか | MVP | β | v1.0 |
|--------|-----|--------------|-----|-----|------|
| ゲーム進捗 | EXP、ストリーク、腹筋ステージ | 単体では非該当の可能性大 | ローカル | ローカル | ローカル＋クラウド（Pro） |
| 実施履歴 | 日付、種目、秒数 | 同上 | ローカル | ローカル | ローカル＋クラウド |
| 端末識別子 | 広告 ID、Analytics ID | **該当** | 収集しない | 同意後のみ | 同意後のみ |
| アカウント | メール、匿名 UID | **該当** | — | — | Pro 同期時 |
| 課金情報 | レシート | **該当**（Store が主体） | — | — | Store 経由。自社は最小保持 |
| 広告・行動データ | 広告 SDK が収集 | **該当** | — | — | CMP 同意後のみ |

### 9.2 フェーズ別 GDPR タスク

| フェーズ | タスク |
|----------|--------|
| **準備** | 収集データ一覧の作成、プライバシーポリシー骨子、第三者 SDK リスト（Analytics・広告・Firebase） |
| **MVP** | ローカル保存のみ。設定に「全データ削除」。外部 SDK は同意なしで個人関連データを送らない |
| **β** | プライバシーポリシー公開、初回 or 分析 ON 時の **オプトイン同意**、同意撤回の導線（設定） |
| **v1.0** | 利用規約、広告 **UMP/CMP**、Pro のデータエクスポート・削除、問い合わせ窓口（メール）、DPA 確認、App Store のプライバシーラベル正確記載 |

### 9.3 アプリ内実装（v1.0 必須）

| 画面 / 機能 | 内容 |
|-------------|------|
| 初回 or 設定 | 「分析の改善に協力」オプトイン（EU はデフォルト OFF） |
| 設定 | プライバシーポリシー・利用規約リンク、同意の変更、全データ削除 |
| 広告表示前 | Google UMP 等で EU ユーザーに広告パーソナライズ同意を取得 |
| Pro アカウント | アカウント削除 → クラウド上の個人データを消去 |
| ストア申請 | App Store / Google Play のデータセーフティフォームを実態と一致させる |

### 9.4 法的文書・運用

| 項目 | 内容 |
|------|------|
| プライバシーポリシー | 収集項目、利用目的、保存期間、第三者提供、ユーザー権利、問い合わせ先 |
| 利用規約 | サブスク、広告、免責、準拠法（EU ユーザーには現地法の権利を妨げない条項） |
| 問い合わせ対応 | データ開示・削除請求への SLA（例：30日以内） |
| データ処理者 | Firebase、AdMob、ストア課金 — 各 DPA・SCC の確認 |
| 代表者 | EU 域内ユーザーが一定数を超える場合は **EU 代理人** の要否を法務で確認 |
